repository is released under Apache-2.0, but PyMuPDF under its free option
is AGPL-3.0, which is a strong copyleft license:

- AGPL obligations attach when the combined work is *distributed* or made
  available to users *over a network*. Purely internal/personal use is
  generally outside its distribution triggers.
- A downstream party who distributes pdf-redact together with PyMuPDF, or
  operates it as a network service (note: the bundled GUI is an HTTP server,
  deliberately bound to 127.0.0.1 only), may need to comply with AGPL-3.0
  terms for the combined work, or obtain Artifex's commercial license.
- This repository itself contains no PyMuPDF code and declares PyMuPDF only
  as a pip dependency; the Apache-2.0 grant here covers only the original
  code in this repository.

If distribution at scale, bundling (e.g. PyPI wheels with vendored
dependencies), or hosted operation is planned, obtain qualified legal advice
and/or contact Artifex (https://artifex.com) about commercial licensing
before doing so.

## Standard library

All other imports (`argparse`, `hashlib`, `re`, `http.server`, `tempfile`,
etc. — the GUI is stdlib plus the user's own browser) are from the Python
standard library, covered by the PSF License Agreement (permissive).
